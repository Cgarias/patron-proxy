Demostración de la Solución: Patrón Proxy Virtual para Carga Diferida

Esta página implementa el patrón Proxy Virtual para la carga de imágenes. Las imágenes grandes solo se cargarán cuando sean visibles en la pantalla del usuario (lazy loading). Esto mejora el tiempo de carga inicial y el uso del ancho de banda.

Abre la consola de tu navegador (F12) y la pestaña "Network" para observar cómo las solicitudes de imágenes se inician solo a medida que haces scroll.

La Solución con el Patrón Proxy Virtual

Utilizamos el patrón Proxy Virtual para gestionar la carga de cada imagen. Aquí, la interfaz (implícita en JS) sería la capacidad de "mostrar" una imagen.

Componentes del Patrón:

  • RealImage (RealSubject): Representa la imagen de alta resolución que realmente se descarga y muestra. Su método display() contiene la lógica de carga real.
  • ImageProxy (Proxy): Actúa como un sustituto de RealImage. Inicialmente, solo sabe la URL de la imagen y muestra un placeholder. Implementa el mismo método display() que RealImage. Cuando display() es invocado por el cliente (a través del `IntersectionObserver`), el Proxy comprueba si la imagen ya ha sido cargada. Si no, crea una instancia de RealImage y le delega la tarea de carga.
  • Cliente: El código que interactúa con ImageProxy. En nuestro caso, el JavaScript que usa el IntersectionObserver para determinar cuándo una tarjeta de imagen entra en el viewport y llama a proxy.display().
